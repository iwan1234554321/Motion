# Motion: Пакет промежуточной анимации для Unity

Библиотека **Motion** представляет собой инструмент для реализации промежуточных анимаций (tweening) в среде Unity с использованием физических алгоритмов и математических кривых. Пакет разработан на основе шаблона проектирования Fluent Interface («текучий интерфейс»), что позволяет декларативно выстраивать цепочки вызовов непосредственно внутри любого компонента `MonoBehaviour`.

---

## Установка

Пакет устанавливается через встроенный менеджер пакетов Unity (Unity Package Manager).

1. Откройте Unity и перейдите в меню **Window** -> **Package Manager**.
2. Нажмите на кнопку **"+"** (плюс) в левом верхнем углу панели.
3. Выберите пункт **"Add package from git URL..."**.
4. Введите следующий URL-адрес репозитория:
   ```text
   https://gitflic.ru/project/iwan1234554321/motion.git
   ```
5. Нажмите кнопку **Add**. После завершения импорта пакет будет готов к работе.

## Основные возможности

* **Физика и кривые**: Поддержка классических анимаций на основе кривых (Tween) и физических анимаций (Spring).
* **Fluent Interface**: Построение понятных и лаконичных цепочек методов.
* **Гибкое управление**: Параллельное выполнение, задержки, циклы и интеграция с асинхронным кодом.
* **Контроль жизненного цикла**: Удобные методы остановки и мгновенного завершения анимаций.

## Руководство по использованию

### 1. Подключение пространства имен
Для начала работы импортируйте библиотеку в ваш скрипт:
```csharp
using Notteam.Motion;
```

### 2. Инициализация анимации
Запуск цепочки методов начинается с расширения для `MonoBehaviour`. Вызовите метод `Motion` через ключевое слово `this`:
```csharp
this.Motion(string id = MotionUtility.DefaultMotionID, bool replace = true)
```
* **`id`**: Уникальный идентификатор анимации. Позволяет управлять конкретным процессом.
* **`replace`**: Флаг замены. Если `true`, то при повторном вызове текущая запущенная анимация с таким же `id` будет остановлена.

*Если параметры не указаны, используются значения по умолчанию (подходит для одиночных анимаций на компоненте).*

### 3. Настройка шагов и компонентов цепочки

После инициализации вы можете комбинировать следующие типы шагов:

* **`AddStep(Action<float> onUpdate, MotionStepSetup setup)`**  
  Добавляет последовательный шаг анимации.  
  * `onUpdate`: Обратный вызов, возвращающий прогресс анимации (значение от 0 до 1).
  * `setup`: Конфигурация шага. Используются производные классы:
    * `MotionStepSetupTween`: Настройка классической анимации по кривой.
    * `MotionStepSetupSpring`: Настройка анимации на основе физики пружины.

* **`AddStepParallel(Action<float> onUpdate, MotionStepSetup setup)`**  
  Автоматически группирует шаги в список для их одновременного (параллельного) воспроизведения. Имеет аналогичные `AddStep` параметры.

* **`AddStepInterval(float duration)`**  
  Добавляет паузу (задержку) заданной длительности в секундах.

* **`AddStepCallback(Action callback)`**  
  Вызывает событие (делегат) в определенной точке цепочки.

#### Общие события жизненного цикла
* **`OnStart(Action callback)`**: Выполняется строго при старте всей цепочки анимации.
* **`OnComplete(Action callback)`**: Выполняется строго при успешном завершении всей цепочки.

### 4. Воспроизведение и асинхронность

Для запуска анимации в конце цепочки вызывается метод `Play`:
```csharp
.Play(Playback playback = Playback.Once, bool unscaledTime = false)
```
* **`playback`**: Режим проигрывания (`Playback.Once`, `Playback.Loop`, `Playback.PingPong`, `Playback.Hold`).
* **`unscaledTime`**: Если `true`, анимация игнорирует системный масштаб времени Unity (`Time.timeScale`).

#### Интеграция с сопрограммами и асинхронными методами
* **Для Coroutines**: Добавьте метод `.Wait()` после `Play()`, чтобы вернуть `IEnumerator`.
  ```csharp
  yield return this.Motion().AddStep(...).Play().Wait();
  ```
* **Для async/await**: Добавьте метод `.Await()` после `Play()`, чтобы вернуть `Task`.
  ```csharp
  await this.Motion().AddStep(...).Play().Await();
  ```

## Остановка и управление состоянием

Для прерывания работы анимаций предусмотрено несколько механизмов:

1. **Через кэшированную ссылку**:  
   Сохраните компонент в переменную и вызовите `Stop()`:
   ```csharp
   motionComponent.Stop();
   ```

2. **Через методы расширения `MonoBehaviour`**:  
   * **`this.StopMotion(string id, bool toDefault = false)`**  
     Останавливает конкретную анимацию по её `id`.  
     * `toDefault = true`: Возвращает анимируемые объекты в исходное положение.  
     * `toDefault = false`: Отключает влияние анимации, оставляя объекты в текущем состоянии.
   * **`this.StopAllMotions(bool toDefault = false)`**  
     Останавливает абсолютно все активные анимации на данном компоненте.
   * **`this.CompleteMotion(string id)`**  
     Мгновенно завершает анимацию с указанным `id`, принудительно устанавливая конечные целевые значения.
